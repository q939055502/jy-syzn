## 计划：配置FastAPI CORS中间件

### 任务目标
在FastAPI应用中添加CORS中间件配置，解决跨域请求和OPTIONS预检请求的405错误问题。

### 具体操作
1. **修改app.py文件**：
   - 导入CORSMiddleware：`from fastapi.middleware.cors import CORSMiddleware`
   - 在创建FastAPI应用实例后，添加CORS中间件配置
   - 配置允许所有来源、所有方法和所有头信息（适合开发环境）

2. **CORS中间件配置内容**：
   ```python
   # 添加CORS中间件
   app.add_middleware(
       CORSMiddleware,
       allow_origins=["*"],  # 允许所有来源，生产环境应限制具体域名
       allow_credentials=True,
       allow_methods=["*"],  # 允许所有HTTP方法，包括OPTIONS
       allow_headers=["*"],  # 允许所有请求头
   )
   ```

3. **配置位置**：在app.py文件中，FastAPI应用实例创建后，路由注册前添加

### 预期结果
- 应用将能正确处理浏览器发送的OPTIONS预检请求
- 跨域请求将能正常进行，不再返回405 Method Not Allowed错误
- 前端登录请求将能成功发送到后端
