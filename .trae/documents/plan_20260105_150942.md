# API文档修复计划

## 1. 问题分析
- API文档中的检测规范响应示例包含不存在的`description`字段
- API文档缺少模型中实际存在的多个字段
- API文档与模型的`to_dict`方法和schemas定义不一致

## 2. 修复内容
更新API文档（API_DOCUMENTATION.md）中3.1.1节的检测规范列表响应示例，使其与以下文件一致：
- 模型文件：`app/models/detection/detection_standard.py` 中的 `to_dict` 方法
- Schemas文件：`app/schemas/detection.py` 中的 `DetectionStandardResponse` 类

## 3. 具体修改
1. **移除不存在的字段**：删除`description`字段
2. **添加实际存在的字段**：
   - `standard_code`：规范编号
   - `standard_type`：规范类型
   - `effective_time`：生效日期
   - `invalid_time`：失效日期
   - `replace_id`：替代的规范ID
   - `remark`：备注信息
3. **保持现有字段**：
   - `standard_id`：规范ID
   - `standard_name`：规范名称
   - `status`：状态
   - `create_time`：创建时间
   - `update_time`：更新时间

## 4. 预期结果
API文档的响应示例与实际实现完全一致，准确反映了检测规范的所有字段，避免给API使用者带来困惑。
