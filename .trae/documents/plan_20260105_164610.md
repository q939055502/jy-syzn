# 修改删除规范返回方式计划

## 1. 目标
将删除检测规范的返回方式修改为与删除用户保持一致，返回200状态码和包含message的JSON响应。

## 2. 修改内容

### 2.1 修改路由文件
- **文件**：`app/routes/detection/standards.py`
- **修改点**：删除规范的路由处理函数

### 2.2 具体修改
1. **移除status_code=204**：使用默认的200状态码
2. **添加ResponseModel返回**：返回包含message的JSON响应
3. **保持错误处理逻辑**：保持现有的错误处理不变

## 3. 预期效果
- 删除规范成功时返回：
  ```json
  {
    "code": 200,
    "message": "删除检测规范成功",
    "data": null
  }
  ```
- 与删除用户的返回格式完全一致

## 4. 实现步骤

1. **修改路由装饰器**：移除`status_code=204`
2. **修改返回值**：使用`ResponseModel`返回格式
3. **保持错误处理**：保持现有的错误处理逻辑不变

## 5. 代码示例

修改前：
```python
@router.delete("/standards/{standard_id}", status_code=204)
def delete_standard(standard_id: int):
    # ... 处理逻辑 ...
    return None
```

修改后：
```python
@router.delete("/standards/{standard_id}", response_model=ResponseModel)
def delete_standard(standard_id: int):
    # ... 处理逻辑 ...
    return ResponseModel(message="删除检测规范成功", data=None)
```

## 6. 测试要点

- 删除规范成功时返回200状态码
- 返回包含message的JSON响应
- 错误处理逻辑保持不变
- 与删除用户的返回格式一致
