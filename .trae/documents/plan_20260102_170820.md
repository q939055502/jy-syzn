## 实现查询账号是否存在的API接口

### 1. 接口设计

**接口路径**: `GET /api/admin/users/check-username`
**请求参数**: 
- `username`: 要检查的用户名（查询参数）
**响应格式**: 
```json
{
  "code": 200,
  "message": "用户名检查成功",
  "data": {
    "exists": true/false
  }
}
```

### 2. 实现步骤

**步骤1**: 在 `app/routes/admin.py` 中添加新的路由函数
- 保持与现有接口一致的代码风格
- 使用 `@router.get` 装饰器
- 应用管理员权限验证
- 实现简单的用户名查询逻辑

**步骤2**: 确保权限控制正确
- 使用 `get_current_admin` 依赖
- 限制只有管理员可以访问该接口

**步骤3**: 实现查询逻辑
- 调用现有的 `UserService.get_user_by_username` 方法
- 根据查询结果返回 `exists` 状态

### 3. 代码实现

```python
@router.get("/users/check-username", response_model=ResponseModel, summary="检查用户名是否存在")
def check_username(
    username: str,
    current_admin: User = Depends(get_current_admin),
    db_and_redis: tuple = Depends(get_db_and_redis)
):
    """
    检查用户名是否已存在（仅管理员）
    
    - **username**: 要检查的用户名
    """
    db, redis = db_and_redis
    from app.services.user.user_service import UserService
    user = UserService.get_user_by_username(username)
    return ResponseModel(
        data={"exists": user is not None},
        message="用户名检查成功"
    )
```

### 4. 预期效果

- 管理员可以通过该接口检查用户名是否存在
- 返回清晰的 JSON 响应，包含 `exists` 字段
- 保持了与现有接口一致的代码风格和层次结构
- 权限控制正确，只有管理员可以访问

### 5. 测试计划

- 调用接口检查不存在的用户名，预期返回 `exists: false`
- 调用接口检查已存在的用户名，预期返回 `exists: true`
- 未授权用户调用接口，预期返回 401 或 403 状态码
