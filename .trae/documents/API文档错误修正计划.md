## API文档错误修正计划

### 1. 统一响应格式修正
- **问题**：文档中部分接口响应示例缺少统一响应格式包装
- **修正**：所有接口响应示例均应包含 `code`、`message` 和 `data` 字段，列表接口额外包含 `total` 字段

### 2. 认证相关接口修正

#### 2.1 获取Token接口 (`/api/auth/token`)
- **问题**：文档中请求格式为 `application/x-www-form-urlencoded`，实际代码使用JSON格式
- **修正**：将请求格式修改为 `application/json`
- **问题**：响应示例缺少统一响应格式包装
- **修正**：添加 `ResponseModel` 包装，包含 `access_token`、`token_type`、`refresh_token` 和 `user` 字段

#### 2.2 刷新Token接口 (`/api/auth/refresh`)
- **问题**：文档中使用 `Authorization` 头传递refresh_token，实际代码使用请求体
- **修正**：修改为使用JSON请求体，字段名为 `refresh_token`
- **问题**：响应示例缺少统一响应格式包装
- **修正**：添加 `ResponseModel` 包装

#### 2.3 获取当前用户信息接口 (`/api/auth/me`)
- **问题**：响应示例中包含 `email` 字段，实际代码不返回该字段
- **修正**：移除响应示例中的 `email` 字段

### 3. 检测相关接口修正

#### 3.1 获取检测指南接口 (`/api/detection/guides/{guide_id}`)
- **问题**：缺少响应示例
- **修正**：添加完整的响应示例

#### 3.2 分类管理接口
- **问题**：文档中路径为 `/api/detection/categories`，实际代码中存在该接口但未在文档中完整描述
- **修正**：完善分类管理接口的描述

### 4. 管理相关接口修正

#### 4.1 角色管理接口
- **问题**：文档中描述了角色管理接口，但实际代码中未找到相关实现
- **修正**：移除或标记为未实现的接口

### 5. 请求示例修正
- **问题**：JavaScript示例中使用 `response.data.code` 判断状态，但实际代码中 `code` 是顶级字段
- **修正**：修正示例代码，使用 `response.code` 或 `response.data.code` 取决于实际实现

- 确保所有接口的请求方法、路径、参数和响应格式与实际代码一致
